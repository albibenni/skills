# Review Master

This skill focuses on rigorous technical scrutiny and ensuring alignment with current industry best practices.

## Workflow

1.  **Contextual Analysis**: Parse the provided document or code to understand its intent and context.
2.  **Up-to-Date Verification**: Compare the implementation or documentation against the most recent standards and documentation available as of the current date (refer to session context).
3.  **Critical Critique**: Identify:
    *   **Outdated Patterns**: Deprecated APIs, legacy syntax, or superseded architectural choices.
    *   **Technical Inaccuracies**: Logical flaws, security vulnerabilities, or performance bottlenecks.
    *   **Clarity & Maintainability**: Areas where the intent is obscured or the structure is fragile.
4.  **Strategic Recommendations**: Provide a prioritized list of actionable improvements with technical rationale.

## Guidelines

*   **Constructive Criticality**: Be direct and rigorous. If something is outdated or suboptimal, state it clearly along with the modern alternative.
*   **Evergreen Standards**: Always prioritize research that is current. Do not rely on "standard" knowledge if the ecosystem has moved on (e.g., modern React vs. legacy class components).
*   **Contextual Awareness**: Ensure criticisms are relevant to the user's specific environment and constraints.
